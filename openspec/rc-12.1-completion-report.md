**Date:** 2026-07-05

## Executive Summary

RC-12.1 replaced the temporary Sponsor Passport portfolio representation with persistent Sponsor Portfolio infrastructure while preserving the RC-10.2 Sponsor Passport contract and Sponsor UI behavior.

The portfolio source moved from `SPONSOR_PASSPORT_PORTFOLIO_JSON` and allowlist helpers to persistent Supabase/Postgres tables exposed through a repository/read-model layer. Evidence Core remains the source of truth for claims, evidence, provenance, audit history, recommendations, and capability data. Portfolio membership is now a sponsor-owned access scope, not an Evidence Core fact.

**Recommended verdict:** RC-12.1 is ready to close after acceptance of this report.

## 2. Architecture Before and After

### Before RC-12.1

```
Sponsor request
   -> Sponsor API
   -> PassportStore
   -> EvidenceCorePassportStore
   -> portfolio allowlist helpers
   -> SPONSOR_PASSPORT_PORTFOLIO_JSON / test override
   -> Evidence Core reads
   -> RC-10.2 Sponsor Passport DTOs
```

### After RC-12.1

```
Sponsor request
   -> Sponsor API
   -> PassportStore
   -> EvidenceCorePassportStore
   -> SponsorPortfolioRepository
   -> sponsor_portfolios / sponsor_portfolio_memberships
   -> Evidence Core reads
   -> RC-10.2 Sponsor Passport DTOs
```

### Boundary Changes

| Concern | Before | After |
|---|---|---|
| Membership source | Env JSON/test override | Persistent database tables |
| Access check | Allowlist helper | Repository membership query |
| Portfolio metadata | Embedded allowlist fields | Persistent membership read model |
| Adapter dependency | Temporary provider | `SponsorPortfolioRepository` interface |
| Contract | RC-10.2 | Unchanged |
| UI | Existing Sponsor UI | Unchanged |
| Evidence Core | Claims/evidence/audit/provenance owner | Unchanged |

## 5. Architectural Decisions

| Decision | Rationale |
|---|---|
| Use SQL/Supabase migrations, not Prisma | The repository has no Prisma setup; existing persistence is SQL/Supabase. Introducing Prisma would have changed the stack outside RC-12.1 scope. |
| Treat portfolio as access scope | Portfolio membership controls sponsor access to institutions; it does not own claims, evidence, confidence, history, or provenance. |
| Preserve RC-10.2 | The source of portfolio membership changed, not the public response shape. |
| Preserve KUX-008 | Sponsor UI behavior remains consumer-equivalent. |
| Keep repository behind `PassportStore` | API routes remain independent of data-source internals. |
| Adapter consumes repository interface | Adapter mappers no longer know about temporary providers or concrete repository construction. |
| Preserve explicit mock mode | `SPONSOR_PASSPORT_DATA_SOURCE=mock` remains supported for local/demo/contract parity and is not considered temporary Portfolio infrastructure. |
| Direct persistent path, no long-lived double read | Production runtime should have one authoritative portfolio source after RC-12.1 closure. |
