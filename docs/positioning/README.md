# Kadarn Positioning

> **Kadarn is the operating system for biospecimen and clinical data programs.**
>
> No es un marketplace. No es un broker. No es un CRO. No es un biobanco.
> Es la infraestructura que permite que todos ellos trabajen juntos.

## 1. The Analogy

Así como **Shopify** es el sistema operativo del comercio electrónico y **Stripe** el sistema operativo de los pagos, **Kadarn** es el sistema operativo de las redes de adquisición y ejecución de programas de biospecímenes.

| Platform | Domain |
|----------|--------|
| Shopify | E-commerce operations |
| Stripe | Payment infrastructure |
| **Kadarn** | Biospecimen & clinical data programs |

Kadarn no vende muestras. No ejecuta ensayos clínicos completos. Opera la infraestructura que permite que esos programas ocurran.

## 5. The Five Core Assets

El verdadero activo de Kadarn no es el software. No son las muestras. No es la red de sitios.

Es una combinación de **cinco elementos** que, juntos, son muy difíciles de replicar:

### Asset 1: Qualified Organization Network

> Una red de organizaciones calificadas (sitios, hospitales, laboratorios y biobancos).

No cualquier organización puede participar. La red está compuesta por entidades que cumplen con estándares definidos de calidad, capacidad y compliance. La calificación es un proceso continuo, no un evento único.

### Asset 2: Reusable Regulatory Library

> Una biblioteca regulatoria reutilizable (protocolos marco, ICFs, SOPs, plantillas y flujos).

Cada programa genera activos regulatorios que pueden ser reutilizados: master protocols, informed consent frameworks, SOPs, regulatory submission templates. Con el tiempo, esta biblioteca reduce drásticamente el tiempo y costo de iniciar nuevos programas.

### Asset 3: Standardized Capability Model

> Un modelo estandarizado de capacidades, donde cada organización publica qué puede hacer, con qué calidad y bajo qué SLA.

Cada organización en la red tiene un perfil de capacidades (tipo de muestras, volúmenes, condiciones de almacenamiento, certificaciones, plazos de entrega). Esto permite matching algorítmico entre necesidades de investigación y capacidades de la red.

### Asset 4: Orchestration Engine

> Un motor de orquestación, capaz de transformar una necesidad científica en un programa ejecutable.

El motor recibe una necesidad de investigación (enfermedad, tipo de muestra, cantidad, región, timeline) y genera un programa: selecciona sitios, asigna laboratorios, establece SLAs, configura el flujo regulatorio, activa la logística y monitorea la ejecución.

### Asset 5: Operational Data

> Datos operacionales acumulados, que permitan responder preguntas como:
> - ¿Quién cumple mejor?
> - ¿Quién entrega más rápido?
> - ¿Qué red es óptima para determinado proyecto?
> - ¿Cuál es la probabilidad de éxito?

Cada programa ejecutado genera datos operacionales que mejoran el sistema. Con el tiempo, Kadarn puede predecir tiempos de entrega, identificar cuellos de botella, optimizar la selección de proveedores y ofrecer pricing basado en datos reales.

## 7. Strategic Implications

### 7.1 Platform Core vs Service Layer (see ADR-001)

Applying the Platform Neutrality Principle:

| Capability | Core (neutral) | Service (specific) |
|------------|---------------|-------------------|
| Organization registry | ✅ Any org type | ❌ |
| Capability model | ✅ Any capability vocabulary | ❌ |
| Discovery/search | ✅ Any biospecimen type | ❌ |
| Regulatory library | ✅ Template engine | ❌ Content (per-program) |
| Program orchestration | ✅ Workflow engine | ❌ Specific workflows |
| Escrow | ✅ Payment rail | ❌ Pricing models |
| Analytics | ✅ Metrics pipeline | ❌ Specific dashboards |

### 7.2 Network Effects

Kadarn's value grows with:

| Side | Growth driver |
|------|--------------|
| More sites | More capacity, better geographic coverage |
| More labs | More processing options, competitive pressure |
| More sponsors | More programs, more data, more revenue for sites |
| More programs | More operational data, better predictions |
| More regulatory templates | Faster program starts, lower friction |

### 7.3 Competitive Moat

The five assets compound over time:

1. **Network**: each new org makes the network more valuable
2. **Regulatory library**: each program reduces cost for the next
3. **Capability data**: more orgs → better matching → better outcomes
4. **Orchestration**: more programs → more refined algorithms
5. **Operational data**: more data → better intelligence → harder to replicate

After 50+ programs, a competitor would need years to catch up.
